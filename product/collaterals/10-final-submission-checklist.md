# Final Submission Checklist

## Product

- [ ] One-liner finalized.
- [ ] Value proposition finalized.
- [ ] Demo path uses Tension Headache.
- [ ] Period Discomfort and Neck & Shoulder Tension are secondary entries.
- [ ] No feature claims full medical diagnosis.

## App

- [ ] Home loads.
- [ ] Three entries are visible.
- [ ] Safety screen appears before routine.
- [ ] Routine preview appears.
- [ ] Camera screen opens.
- [ ] At least one feedback state works.
- [ ] Timer/progress works or can be simulated.
- [ ] Recap appears.
- [ ] Fallback/demo mode works.

## Safety

- [ ] UI does not say "treat".
- [ ] UI does not say "diagnose".
- [ ] UI does not say "cure".
- [ ] UI does not say "detect disease".
- [ ] Pregnancy caution included where relevant.
- [ ] Red-flag stop copy included.

## Demo

- [ ] 90-second script practiced.
- [ ] 30-second script practiced.
- [ ] Fallback script practiced.
- [ ] Golden path ran 3 times.
- [ ] Demo video recorded.
- [ ] Backup recording saved.

## Devpost

- [ ] Final copy pasted.
- [ ] GitHub URL added.
- [ ] Demo URL added.
- [ ] Built With section updated.
- [ ] Screenshots added.
- [ ] Video added.
- [ ] Safety boundary visible in description.
- [ ] No medical treatment claims.

## Judge Q&A

- [ ] Core value answer prepared.
- [ ] Medical product answer prepared.
- [ ] ChatGPT comparison answer prepared.
- [ ] Accuracy answer prepared.
- [ ] Market opportunity answer prepared.
- [ ] Future work answer prepared.

## Final Product Owner Sign-Off

Sign off only when:

- [ ] A non-technical person can understand the product in 30 seconds.
- [ ] A technical judge can see AI/vision relevance.
- [ ] A safety-conscious judge can see the non-diagnostic boundary.
- [ ] The live demo has a fallback.
- [ ] The team knows what not to say.
