# Product Owner Plan

## Product Owner 职责

Product Owner 的目标是让团队不迷失在功能堆叠里，保证项目始终围绕一个可以赢的 demo 展开。

Product Owner 不负责写全部代码，但要负责以下决策：

| 领域 | 责任 |
|---|---|
| 产品定位 | 明确产品不是 AI doctor，而是 guided self-care coach |
| 用户场景 | 选择最适合 demo 的用户痛点和主路径 |
| Scope 控制 | 决定做什么、不做什么、砍什么 |
| 文案 | 入口、safety、routine、recap、pitch 统一口径 |
| Demo | 保证 90-120 秒演示能讲清楚价值 |
| Devpost | 准备标题、描述、价值、技术、挑战和未来方向 |
| 风险 | 避免医疗诊断、治疗承诺和过度 claims |

## 48 小时优先级

### P0: 必须完成

- [ ] 一句话 value prop 定稿。
- [ ] 主 demo 选择为 tension headache。
- [ ] 三个入口文案完成。
- [ ] Safety screen 文案完成。
- [ ] Demo script 完成。
- [ ] Devpost 草稿完成。
- [ ] Judge Q&A 完成。
- [ ] Product acceptance checklist 跑一遍。

### P1: 有时间再做

- [ ] 加一版更 emotional 的 period discomfort story。
- [ ] 加一版更 technical 的 AI/vision explanation。
- [ ] 准备 30 秒、90 秒、2 分钟三个 pitch 版本。
- [ ] 准备 booth/sponsor 对话版本。

### P2: 不做

- [ ] 不做复杂中医辨证。
- [ ] 不写临床疗效证明。
- [ ] 不做全身 AR 或 3D body model 的产品承诺。
- [ ] 不把 symptom self-report 解释成疾病判断。

## Product Decision Log

| Decision | Alternatives | Reason |
|---|---|---|
| 只做手部 | 全身 AR、肩颈姿势、问诊 App | 手部最容易做视觉反馈，scope 最稳 |
| 主 demo 用 tension headache | 经期不适、肩颈紧张 | 头痛场景更安全、手部示范清楚、讲解快 |
| 产品定位为 wellness coach | AI doctor、TCM diagnosis app | 降低医疗风险，价值更清楚 |
| Routine 内容固定 | LLM 自由推荐穴位 | 避免幻觉和医疗内容失控 |
| AI 放在教练语言和 safety | AI 决策治疗方案 | 更适合 hackathon，也更可信 |

## 每 2 小时 Product Owner 检查

- [ ] Golden path 还能不能跑？
- [ ] 当前功能是否支持 demo 主张？
- [ ] 是否出现医疗诊断或治疗承诺？
- [ ] 是否有比当前功能更该砍的东西？
- [ ] Demo script 是否需要随实现调整？
