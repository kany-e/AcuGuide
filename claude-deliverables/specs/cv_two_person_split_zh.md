# 计算机视觉特征提取：两人分工（简化版）

## 核心结论
**不要按"功能"分工**（一个人做"按压计数"，一个人做"画圈识别"）——因为所有功能都读同一份手部关键点数据，会互相冲突、重复造轮子。

**应该按"流程层"分工**，中间用一个固定的数据格式（`FrameState`）作为交接口。

```
   A 负责：摄像头 → 关键点 → 朝向 → 目标点 → 是否按对
                              │
                        FrameState（交接口）
                              ▼
   B 负责：时间序列 → 动作识别 → 计数 → 状态机 → 教练提示语
```

---

## A 的工作：感知 / 几何（上游）
把"画面"变成"每一帧的干净状态"。

- 启动 MediaPipe，追踪两只手（按压手 + 被按手）
- 关键点**去抖动**（让 B 拿到干净信号）
- 判断**手心 / 手背**朝向，处理左右手镜像
- 读取 `acuguide_hand_points.json`，算出**目标点位置 + 允许范围**
- 判断**指尖有没有按到点**，以及用的是**指尖 / 指腹 / 指根**
- 输出：每帧一个 `FrameState`

## B 的工作：时间 / 教练（下游）
把"每帧状态的序列"变成"动作和反馈"。

- 缓存最近约 2 秒的帧
- **动作识别**：稳按 / 画圈 / 反复按压
- **按压次数**：对深度信号做波峰检测
- **稳定度、节奏**（给总结页用）
- **状态机**：没手 → 朝向错 → 找位置 → 按到但不稳 → 稳按计时 → 暂停 → 完成
- 选出**教练提示语**（可接 LLM 改写）

---

## 交接口：`FrameState`（第一小时就定死）
A 每帧产出一个，B 只看这个、永远不碰原始关键点。

```json
{
  "t": 1234.56,
  "receivingHand": { "present": true, "face": "dorsal", "handSize": 0.18 },
  "pressingFinger": { "present": true, "contactPart": "tip" },
  "target": { "id": "TE3", "xy": [0.49, 0.60], "toleranceR": 0.029 },
  "contact": { "onTarget": true, "offset_xHandSize": 0.07, "depthProxy": 0.42 },
  "quality": { "confidence": 0.91 }
}
```

---

## 为什么能真正并行
1. **第一小时**：两人一起把 `FrameState` 格式定下来，A 手写 2–3 个假数据文件（手出现、漂移、稳按、离开）。
2. **中间阶段**：A 做真实识别，B 用假数据做整套逻辑——**互不等待**。
3. **整合**：把 B 读假数据的口子换成 A 的实时流。格式没变就能直接跑通。

> B 甚至可以用**合成信号**单独测试：正弦波 → 应判定"画圈"；方波脉冲 → 应数出 N 次按压。完全不用摄像头。

---

## 我的几条改进建议
1. **录制+回放工具（最有用）**：A 从第一天就把真实片段录成文件。既方便调试，也是**演示时摄像头出问题的备用方案**（直接放录像）。
2. **接口定死并加版本号**：两人分工最容易死在"接口偷偷改了"。
3. **所有去噪放在 A 这边**：B 完全信任拿到的信号，少很多"是你 bug 还是我 bug"。
4. **脆弱功能加开关**：按压计数、力度这类摄像头测不准的，设成可关闭；**核心流程 = 按到 + 稳住 + 计时**（最可靠），整合来不及也有完整 demo。
5. **尽早跑通一条最简单的整条链路**：手检测到 → 圆环填满 → 完成。先证明 A→B→UI 通了，再加功能。

---

## 什么时候干脆别拆 CV
你们 PRD 里本来就只设了**一个** Vision/AI 负责人。这套 CV 流程不大、又高度耦合，硬塞两个人进去，沟通成本可能大于收益。

如果两人 CV 经验不均，**更好的拆法是按"CV vs 应用"分**：
- **第一人**：整条 CV 流程（A+B 一起）
- **第二人**：App 界面、摄像头页、总结页，以及消费 `CoachState` 的 LLM 教练层

**口诀**：两人都懂信号/CV 才在 CV 内部拆；否则就沿"CV 管线 vs 应用层"这条线拆，中间照样用 `FrameState` / `CoachState` 两个接口。
